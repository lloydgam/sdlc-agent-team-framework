================================================================================
                    EXECUTION SUMMARY - HOW TO RUN YOUR PROJECT
================================================================================

Once you have your project details ready (brief, requirements, task list),
follow these steps to execute your project using the Agent Framework.

================================================================================
QUICK OVERVIEW
================================================================================

The execution process has 9 main phases:

1. PREPARE PROJECT FILES
   - Create PROJECT_BRIEF.md (what are you building?)
   - Create REQUIREMENTS.md (what features do you need?)
   - Create TASK_LIST.json (what tasks need to be done?)

2. ORGANIZE PROJECT FOLDER
   - Copy agent files to your project
   - Setup shared resources folder
   - Organize all files

3. SPAWN TEAM LEAD AGENT
   - Run: SPAWN_AGENT: TEAM_LEAD
   - Team Lead analyzes requirements
   - Team Lead decomposes into tasks
   - Team Lead identifies dependencies

4. SPAWN SPECIALIZED AGENTS
   - Spawn Backend Developer
   - Spawn Frontend Developer
   - Spawn QA Engineer
   - Spawn DevOps Engineer
   - Spawn Documentation Specialist

5. PARALLEL EXECUTION
   - All agents work in parallel
   - Each claims tasks from shared list
   - Each executes their domain work
   - Each updates progress regularly

6. TEAM LEAD MONITORING
   - Monitor task list regularly
   - Identify blockers
   - Escalate issues
   - Support team

7. BLOCKER RESOLUTION
   - Agents report blockers
   - Team Lead investigates
   - Team Lead coordinates fix
   - Agents resume work

8. QUALITY ASSURANCE
   - QA validates work
   - Issues identified
   - Development team fixes
   - QA verifies fixes

9. DEPLOYMENT & COMPLETION
   - Deploy to production
   - Monitor for issues
   - Document lessons learned
   - Celebrate success

================================================================================
STEP-BY-STEP EXECUTION
================================================================================

PHASE 1: PREPARE PROJECT FILES
─────────────────────────────────────────────────────────────────────────────

Step 1.1: Create PROJECT_BRIEF.md
  What to include:
  - Project overview (2-3 sentences)
  - Goals (3-5 bullet points)
  - Success criteria (3-5 bullet points)
  - Scope (what's included/excluded)
  - Team (who's involved)
  - Key milestones (major phases)

Step 1.2: Create REQUIREMENTS.md
  What to include:
  - Functional requirements (features)
  - Non-functional requirements (performance, security, scalability)
  - Technical constraints (tech stack, platforms, integrations)
  - Quality standards (test coverage, code quality, documentation)

Step 1.3: Create TASK_LIST.json
  What to include:
  - Task ID (TASK-001, TASK-002, etc.)
  - Title (clear, specific)
  - Description (what needs to be done)
  - Assignee (which agent: BACKEND_DEVELOPER, FRONTEND_DEVELOPER, etc.)
  - Status (TODO initially)
  - Priority (HIGH, MEDIUM, LOW)
  - Dependencies (which tasks must be done first)
  - Completion criteria (how do you know it's done?)

PHASE 2: ORGANIZE PROJECT FOLDER
─────────────────────────────────────────────────────────────────────────────

Create this structure:
  your-project/
  ├── PROJECT_BRIEF.md
  ├── REQUIREMENTS.md
  ├── TASK_LIST.json
  ├── agents/
  │   ├── team-lead-agent.md
  │   ├── backend-agent.md
  │   ├── frontend-agent.md
  │   ├── qa-agent.md
  │   ├── devops-agent.md
  │   ├── docs-agent.md
  │   └── agent-framework.js
  └── shared/
      ├── task-list.json
      └── communication-log.json

Copy agent files from the framework to your project.

PHASE 3: SPAWN TEAM LEAD AGENT
─────────────────────────────────────────────────────────────────────────────

In Claude Code, run:

  SPAWN_AGENT: TEAM_LEAD

  Context:
  - Read PROJECT_BRIEF.md
  - Read REQUIREMENTS.md
  - Review TASK_LIST.json
  - Use agents/agent-framework.js

  Instructions:
  1. Analyze project requirements
  2. Verify task decomposition
  3. Identify dependencies
  4. Update shared/task-list.json
  5. Prepare to spawn agents

Team Lead will:
  ✅ Analyze your project
  ✅ Verify task decomposition
  ✅ Identify dependencies
  ✅ Update task list
  ✅ Prepare for agent spawning

PHASE 4: SPAWN SPECIALIZED AGENTS
─────────────────────────────────────────────────────────────────────────────

Spawn each agent in Claude Code:

Backend Developer:
  SPAWN_AGENT: BACKEND_DEVELOPER
  Context: PROJECT_BRIEF.md, REQUIREMENTS.md, TASK_LIST.json
  Task: Claim backend tasks and execute

Frontend Developer:
  SPAWN_AGENT: FRONTEND_DEVELOPER
  Context: PROJECT_BRIEF.md, REQUIREMENTS.md, TASK_LIST.json
  Task: Claim frontend tasks and execute

QA Engineer:
  SPAWN_AGENT: QA_ENGINEER
  Context: PROJECT_BRIEF.md, REQUIREMENTS.md, TASK_LIST.json
  Task: Claim QA tasks and execute

DevOps Engineer:
  SPAWN_AGENT: DEVOPS_ENGINEER
  Context: PROJECT_BRIEF.md, REQUIREMENTS.md, TASK_LIST.json
  Task: Claim DevOps tasks and execute

Documentation Specialist:
  SPAWN_AGENT: DOCUMENTATION_SPECIALIST
  Context: PROJECT_BRIEF.md, REQUIREMENTS.md, TASK_LIST.json
  Task: Claim documentation tasks and execute

PHASE 5: PARALLEL EXECUTION
─────────────────────────────────────────────────────────────────────────────

All agents work in parallel:
  - Each agent checks task list
  - Each agent claims available tasks
  - Each agent executes their work
  - Each agent updates progress regularly
  - Each agent reports blockers immediately

Communication:
  All agents ←→ Shared Task List ←→ Team Lead

PHASE 6: TEAM LEAD MONITORING
─────────────────────────────────────────────────────────────────────────────

Team Lead continuously:
  - Checks task list regularly
  - Identifies blockers
  - Escalates issues
  - Adjusts priorities
  - Supports team

Monitor these metrics:
  ✅ Task completion percentage
  ✅ Progress updates
  ✅ Blocker count
  ✅ Quality metrics
  ✅ Velocity

PHASE 7: BLOCKER RESOLUTION
─────────────────────────────────────────────────────────────────────────────

When blocker occurs:
  1. Agent reports blocker to task list
  2. Team Lead investigates
  3. Team Lead coordinates fix
  4. Blocker resolved
  5. Agent resumes work
  6. Task list updated

PHASE 8: QUALITY ASSURANCE
─────────────────────────────────────────────────────────────────────────────

QA Agent validates:
  - Write test cases
  - Execute tests
  - Identify issues
  - Verify fixes

Quality gates:
  ✅ Test coverage > 80%
  ✅ No critical bugs
  ✅ Performance acceptable
  ✅ Security audit passed
  ✅ Documentation complete

PHASE 9: DEPLOYMENT & COMPLETION
─────────────────────────────────────────────────────────────────────────────

Pre-deployment:
  ✅ All tasks marked DONE
  ✅ Quality gates passed
  ✅ Tests passing
  ✅ Documentation complete

Deployment:
  1. DevOps prepares deployment
  2. All agents verify readiness
  3. Deploy to production
  4. Monitor for issues

Completion:
  - All tasks complete
  - Quality gates passed
  - Deployed to production
  - Lessons learned documented
  - Celebrate success! 🎉

================================================================================
MONITORING & TRACKING
================================================================================

Daily Checklist:
  ✅ Check task list for updates
  ✅ Review progress percentage
  ✅ Identify blockers
  ✅ Check quality metrics
  ✅ Review communication log
  ✅ Support team
  ✅ Update status

Key Metrics:
  - Velocity: Tasks completed per period
  - Quality: Test coverage, defect count
  - Timeline: Progress percentage, remaining work
  - Blockers: Count, resolution time
  - Satisfaction: Team feedback

Progress Tracking:
  View task list: cat shared/task-list.json | jq '.'
  View blockers: cat shared/task-list.json | jq '.tasks[] | select(.status == "BLOCKED")'
  View communications: cat shared/communication-log.json | tail -20

================================================================================
TROUBLESHOOTING
================================================================================

Issue: Agent Not Responding
  Solution:
  1. Check task list for last update
  2. Send direct message to agent
  3. Escalate to Team Lead if no response
  4. Reassign task if needed

Issue: Blocker Not Resolved
  Solution:
  1. Verify blocker is documented
  2. Escalate to Team Lead
  3. Investigate root cause
  4. Coordinate resolution
  5. Resume work when resolved

Issue: Quality Issues
  Solution:
  1. QA reports issues immediately
  2. Development team fixes issues
  3. Increase testing frequency
  4. Implement quality gates
  5. Prevent similar issues

Issue: Timeline Slipping
  Solution:
  1. Identify root causes
  2. Assess impact
  3. Adjust timeline if needed
  4. Increase resources if possible
  5. Reduce scope if necessary
